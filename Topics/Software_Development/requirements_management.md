# Software Requirements Management

## Types of Requirements
- **Functional**: What system should do
- **Non-functional**: Performance, security, usability
- **Business**: Stakeholder needs
- **User**: End-user expectations
- **System**: Technical constraints

## Elicitation Techniques
- Interviews, surveys, workshops
- Use cases, user stories
- Prototyping, observation

## Specification
- **SRS (Software Requirements Specification)**: Detailed document
- **Use Case Diagrams**: Actors, scenarios
- **Acceptance Criteria**: Definition of Done

## Validation
- Reviews, inspections
- Prototypes, simulations
- Traceability matrices

## Management
- **Version Control**: Git for requirements
- **Change Control**: Impact analysis
- **Prioritization**: MoSCoW (Must, Should, Could, Won't)
- **Tools**: Jira, Confluence, ReqIF
