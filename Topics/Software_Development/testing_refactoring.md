# Software Testing and Refactoring

## Testing Types
- **Unit Testing**: Individual components (JUnit, pytest)
- **Integration Testing**: Module interactions
- **System Testing**: End-to-end
- **Acceptance Testing**: User requirements
- **Regression Testing**: After changes
- **Performance Testing**: Load, stress, scalability

## Testing Techniques
- **Black-box**: Input/output, equivalence partitioning, boundary value
- **White-box**: Code coverage, statement/branch/path
- **Gray-box**: Partial knowledge

## Test-Driven Development (TDD)
- Write test first, then code
- Red-Green-Refactor cycle

## Refactoring
- **Definition**: Improve code without changing behavior
- **Techniques**: Extract method, rename, move class
- **Code Smells**: Long methods, duplicate code, large classes
- **Tools**: IDE refactorings, SonarQube

## Automation
- **CI/CD**: Jenkins, GitHub Actions
- **Test Frameworks**: Selenium (UI), Postman (API)

## Metrics
- **Coverage**: Line, branch
- **Defect Density**: Bugs per KLOC
- **Test Effectiveness**: Bugs found vs. escaped
