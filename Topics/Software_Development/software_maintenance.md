# Software Maintenance

## Types
- **Corrective**: Fix bugs
- **Adaptive**: Adapt to new environments
- **Perfective**: Improve performance/features
- **Preventive**: Refactor for future changes

## Process
- **Request**: Issue tracking
- **Analysis**: Impact assessment
- **Design**: Solution planning
- **Implementation**: Code changes
- **Testing**: Regression testing
- **Deployment**: Release

## Challenges
- Legacy code understanding
- Documentation gaps
- Regression risks
- Cost estimation

## Metrics
- **MTTR (Mean Time To Repair)**: Fix speed
- **Availability**: Uptime percentage
- **Defect Rate**: Post-maintenance bugs

## Evolution
- **Reengineering**: Rewrite with modern tech
- **Reverse Engineering**: Understand existing code
