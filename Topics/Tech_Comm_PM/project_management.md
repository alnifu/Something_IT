# Technical Communication & Project Management

## Project Management Methodologies
- **Waterfall**: Sequential phases (Requirements -> Design -> Implementation -> Testing -> Deployment)
- **Agile**: Iterative, flexible (Sprints, user stories)
- **Scrum**: Framework for Agile (Roles: Product Owner, Scrum Master, Team; Events: Sprint Planning, Daily Scrum, Sprint Review, Retrospective)
- **Kanban**: Visualize workflow (Boards with columns like To Do, In Progress, Done)
- **Lean**: Eliminate waste (Focus on value, continuous improvement)

## Key Concepts
- **Scope**: Project deliverables (Work Breakdown Structure - WBS)
- **Time**: Schedule and milestones (Gantt charts, PERT)
- **Cost**: Budget and resources (Earned Value Management - EV, PV, AC)
- **Quality**: Standards and requirements (Quality Assurance vs Quality Control)
- **Risk**: Potential issues and mitigation (Risk Register, Probability/Impact Matrix)
- **Stakeholders**: People affected by project (Stakeholder Analysis, Communication Plan)

## Critical Path Method (CPM)
- **Critical Path**: Longest sequence of dependent tasks that determine project duration
- **Float/Slack**: Amount of time a task can be delayed without affecting project end date
- **Forward Pass**: Calculate earliest start/finish times
- **Backward Pass**: Calculate latest start/finish times
- **Multiplicity**: Handling multiple critical paths or parallel paths

## Crashing
- **Crashing**: Technique to shorten project duration by adding resources (cost increases)
- **Fast Tracking**: Overlap tasks that were sequential
- **Resource Leveling**: Balance resource usage over time

## Communication
- **Active Listening**: Pay attention and respond
- **Clear Writing**: Concise, unambiguous (Use plain language, avoid jargon)
- **Presentations**: Structure (Introduction, Body, Conclusion), visuals (Charts, diagrams), delivery (Practice, engage audience)
- **Feedback**: Constructive criticism (Specific, timely, actionable)
- **Documentation**: Manuals, guides, reports (User docs, technical specs, status reports)
